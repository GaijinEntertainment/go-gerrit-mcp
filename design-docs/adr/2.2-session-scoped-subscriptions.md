# Session-scoped explicit subscriptions over account-wide watching

- **Status:** accepted
- **Date:** 2026-07-13
- **Initiative:** 02-review-notifications

Every prior Gerrit notification integration observed in the field scoped to the account: poll everything the
authenticated user owns, notify about all of it. That design failed twice — concurrent agent sessions woke each
other up about changes they never touched, and unfiltered activity (CI votes) was indistinguishable from human
review. The account is the wrong scope: the unit that cares about a change is the session working on it.

Decision: notification scope is an explicit, per-session subscription set. The agent subscribes to a change (taught
by model-facing prompts to do so after pushing something reviewable), the server polls only subscribed changes over
REST on an interval (default 60s; nothing subscribed, nothing polled), and a per-change cursor turns poll results
into deltas. Subscriptions live in process memory — one stdio server per agent session makes isolation structural
rather than configured — and are trail-free: nothing about them is visible on the Gerrit instance. Subscriptions end
explicitly (unsubscribe tool), or automatically when the change reaches a terminal state (merged/abandoned), with a
final notification announcing the auto-unsubscription. Event filters drop the authenticated account's own activity
by default (flag to include) and apply operator-configured exclusions: accounts (usernames/IDs) and message-content
regexes that fail startup loudly when invalid. No tag-based bot heuristics — whether a bot is noise or signal (a CI
verdict often is the awaited outcome) is the operator's call, not the server's.

Rejected alternatives: account-wide `owner:self` polling (the failure that motivated this design); Gerrit attention
set as the signal (account-scoped, and trusts server-side service-user hygiene); SSH stream-events (a second
credential class plus a capability grant most accounts lack); webhooks (server-admin territory). Consequence
accepted: a restarted session loses its subscriptions and cursors — resuming agents re-subscribe and take one full
comment read, which is the honest trade for having no persistent state.
