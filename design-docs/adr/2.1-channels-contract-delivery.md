# Review notifications ride the Claude Code channels contract

- **Status:** accepted
- **Date:** 2026-07-13
- **Initiative:** 02-review-notifications

Review notifications must reach the model mid-session without a fetch: after subscribing, the agent receives the
activity itself, self-sufficient, with no consumption tool. Standard MCP offers no model-visible push — logging
(`notifications/message`) and resource-updated notifications are client-surface concerns that documented clients do
not inject into the model's context. The Claude Code channels contract (research preview, Claude Code >= 2.1.80) is
exactly this missing path: a stdio MCP server that declares the `claude/channel` experimental capability and emits
`notifications/claude/channel` events, whose `content` the client injects into the session as a `<channel>` block
the model reads and acts on.

Decision: when review notifications are enabled, this server declares the `claude/channel` capability and delivers
every notification as a channel event — rendered llmxml as `content`, routing context (change number, event kind) as
`meta` attributes. There is no fallback delivery mechanism: clients without channel support silently drop the
events, which degrades to exactly the pre-feature behavior. Because the pinned Go SDK exposes no generic
notification sender, emission goes through a wrapping transport that captures the `mcp.Connection` (whose `Write` is
documented concurrency-safe) and writes the ID-less notification directly.

Consequences accepted: coupling to a research-preview contract that may change; during the preview, operators enable
the channel per session (`claude --dangerously-load-development-channels server:<name>` for a bare `.mcp.json`
server) and org policy can block delivery entirely. Rejected alternatives: a pull tool (breaks the self-sufficiency
requirement — the agent would poll, which is the pre-feature status quo with extra steps); MCP logging notifications
(not model-visible in any documented client); resource subscriptions (same visibility problem plus a client
subscription dance).
